Central Data Elements and Messages

General

All three messages contain information about 'declarant'. Declarant shall contain information about authenticated sender in SSNN, either the one logged into the portal and submitting data or the company that sends data to SSNN via RIM.

Special Field Validations

  • at house level, field: additionalInformationCode shall contain a specific string with data consisting of specific code values
    • IIMRE - Release for free circulation
    • IWARE - Placement in customs warehouse
    • ICORE - Collective clearance
    • IDOCR - Documents
    • IATCA - ATA carnet
  • at house level, field: previousReferencedDocumentList, referenceNumber, shall contain a specific string with data consisting of three elements
    • declarantNumber: 9 characters, numeric
    • declarationDate: YYYY-MM-DD
    • sequenceNumber: 3 characters, numeric
  • These three strings of data shall be sent to Norwegian Customs in the referenceNumber field
  • Corresponding type shall be CUDE according to code list.

Criteria for Submission of DOG

As illustrated in the figure above, declaration of goods with message type LIMITED shall not be submitted to Digitoll. The information filled in DOG with type LIMITED will be transmitted to Norwegian Customs through use of relationship messages as explained below.

Special about Message Type DIRECT_CONSIGNMENT_ONLY

When using the message type DIRECT_CONSIGNMENT_ONLY it shall be possible to attach multiple instances of house consignments in the same declaration of goods.

Special Field Validations

  • Relationship message has field for directConsignment (true/false). The logic for setting this field:
    • TRUE - transportContractNumber is retrieved from DOG at house level, messageType: DirectConsignmentOnly (transport document at master does not exist)
    • FALSE - transportContractNumber is retrieved from DOG at master level (applies to the three other message types that all have transport document number at master level)

Order of Message Submission

The Norwegian Coastal Administration shall send the messages in the following order:

  • Transport message
  • DeclarationOfGoods (wait to send relationship message until you have synchronously received ok on submission of DoG)
  • Relationship

For updates:

  • Transport message can be updated without initiating any other submissions
  • When updating a DOG, a new version of relationship shall subsequently be sent (complete content, references to all DOG)
