The solution for notification and disclosure has an API for reporting arrival of transport

Background:

There is a need to create a function that makes it possible to send an arrival message for road transport when the business has transport authorization (Transport authorization for unmanned border crossings - business - Norwegian Customs). These transports cross the border at crossings where Norwegian Customs is not present.

The function is nevertheless not limited to road transports with transport authorization. It is designed to be used for all modes of transport. However, it should not be used without an agreement with Norwegian Customs.

Arrival API:

Arrival messages follow a standard pattern:

  • Create (POST) - The business creates an arrival message and receives a request-id
  • Retrieve response (GET) - The business retrieves the validation status of the arrival message using the request-id

In the arrival message, the MRN, time, and customs office for the arrival of the transport are specified. API documentation is available here: Swagger UI

Case example:

  • The driver or the driver's representative submits an arrival message for the transport in connection with the transport crossing the border.
  • Based on the message status = success, the driver/driver's representative should retrieve the entry status for the transport. The endpoint entry/status/{mrn} (Swagger UI) can be used.
    • If the transport receives valid entry = true, the goods have transitioned to the selected procedure and can be unloaded upon arrival.
    • If the transport receives valid entry = false, Norwegian Customs must be contacted before the goods can be unloaded.
