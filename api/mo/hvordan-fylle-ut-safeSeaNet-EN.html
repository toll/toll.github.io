How to Fill Out Messages from SafeSeaNet

Interface Description

The services are made available as REST services over HTTPS, with data in JSON format. 

POST https://<env>/api/movement/maritime-ssnn/v1/transport
POST https://<env>/api/movement/maritime-ssnn/v1/declaration-of-goods
POST https://<env>/api/movement/maritime-ssnn/v1/relationship

PUT/DELETE https://<env>/api/movement/maritime-ssnn/v1/transport/{MRN}
PUT/DELETE https://<env>/api/movement/maritime-ssnn/v1/declaration-of-goods/{MRN}
PUT https://<env>/api/movement/maritime-ssnn/v1/relationship/{MRN}

GET https://<env>/api/movement/maritime-ssnn/status/v1/transport/validation-status/{requestId}
GET https://<env>/api/movement/maritime-ssnn/status/v1/declaration-of-goods/validation-status/{requestId}
GET https://<env>/api/movement/maritime-ssnn/status/v1/relationship/validation-status/{requestId}

where

  • <env> is replaced with
    • api.toll.no for production
    • api-test.toll.no for playground (test environment)
  • {requestId} is replaced with the reference returned from POST/PUT/DELETE calls
  • {MRN} is replaced with the reference returned from the ".../validation-status..." endpoints.

If you discover security challenges in our solutions, please contact us; see here for info

Special about DELETE and Relationship

Since relationship messages are generated based on declaration-of-goods, it is not logical to delete a declaration-of-goods directly. They are deleted indirectly through declaration-of-goods being changed. Therefore, DELETE on declaration-of-goods is not an option.

If all declaration-of-goods are deleted, an empty relationship message is submitted as PUT.
