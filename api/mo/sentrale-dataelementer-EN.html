For all messages, there will be a set of data that is only related to the message and its submission. This data includes information about the customs representative and identification of the message (references to the message itself). See the section Linking Messages for a detailed description.

The tables below provide an overview of central data elements related to transport and consignments. The detailed information models for transport, master consignments, and consignments are published in the API definitions.

Transport Notification

Area Description
Notify Transport Information about transport and reference to all consignments on board the transport
Responsible for Information Driver of the transport, according to the Customs Act ยง 2-3
Information Provider Customs representative according to the Customs Act ยง 7-21 – Forwarder/Carrier/Consignor
Recipient of Information Norwegian Customs
Is Information Mandatory? Yes
When is Information Desired by Norwegian Customs? As early as possible, but no later than upon arrival at the border
When is Information Update Desired? Information must be updated when important parts of the information change. Typically when changing license plates, driver, time/place of border crossing, or changes in the list of master consignments
Submission Deadline No later than upon border crossing
Important Information Elements

License plate(s) of the transport means and/or route number for scheduled traffic (e.g., scheduled flights)

Information about the driver

Time/place of border crossing

Reference number/waybill number for all master consignments (Master Consignment) and/or consignments (House Consignment) on board the transport
Special about Updating Transport Notification When updating and canceling, the MRN received in the original message is the key to be used to identify the correct transport
